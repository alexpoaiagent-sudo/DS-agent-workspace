# Протокол найма ИИ-агента (Agency Hiring Protocol)

> Запускается на шаге «Исполнитель» в Ритуале согласования.
> Цель: подобрать профильного агента под роль задачи или использовать модель по умолчанию.

---

## Алгоритм (4 шага)

### Шаг 1 — Сформулируй задачу

Чётко определи: что нужно сделать и какой результат получить.

### Шаг 2 — Выбери модель

Sonnet / Haiku / Opus — в зависимости от сложности задачи.

### Шаг 3 — Проверь реестр агентства

Задай вопрос:
> **«Какой ИИ-агент из агентства может выполнить эту задачу?»**

Открой `agency/REGISTRY.md` → найди подходящего агента.

```
Задача                       →  Агент в реестре?
──────────────────────────────────────────────────
Написать регламент кофейни   →  VK Coffee Analyst    ✅
Создать Telegram-бота        →  Code Engineer         ✅
Обновить ГПХ / реквизиты    →  HR Specialist         ✅
Починить скрипт агента       →  Environment Engineer  ✅
Письмо архитектору / парк    →  Park Architect        ✅
Спланировать неделю / РП     →  Strategist            ✅
SEO-текст / Instagram        →  —                     ❌
Финансовая модель            →  —                     ❌
```

### Шаг 4 — Принять решение

**Вариант А — Агент найден:**
```
→ Объявить: «Нанимаю [Имя агента]»
→ Прочитать agency/agents/{slug}.md
→ Работать в роли агента
```

**Вариант Б — Агента нет:**
```
→ Объявить: «Агент не найден — работаю моделью по умолчанию»
→ Зафиксировать в agency/GAPS.md (дата, задача, нужная роль)
→ Работать без специализации

Правило расширения: 3+ записи одной роли в GAPS → создать нового агента
```

**Создание нового агента (при 3+ в GAPS):**
```
→ Создать agency/agents/{slug}.md по шаблону
→ Добавить в REGISTRY.md
→ Очистить строки из GAPS.md → архив
→ Нанять при следующей задаче
```

---

## Формула объявления в Ритуале

```
Модель: [Sonnet / Haiku / Opus]
Агент:  [Имя из реестра] / [создаём нового] / [по умолчанию]
```

**Пример А — агент найден:**
> Модель: Sonnet. Агент: **Code Engineer** — нанимаю, загружаю контекст.

**Пример Б — создаём нового:**
> Модель: Sonnet. Агент для Instagram-контента — **нет в реестре, создаю нового агента** перед началом работы.

**Пример В — по умолчанию:**
> Модель: Sonnet. Агент: **нет в реестре → работаю по умолчанию**, фиксирую пробел.

---

## Правило расширения реестра

> Если одна и та же роль встречается **3+ раза** в GAPS.md → создать нового агента.

Алгоритм создания:
1. Определить домен и специализацию
2. Создать `agency/agents/{slug}.md` по шаблону
3. Добавить в `REGISTRY.md`
4. Убрать из GAPS.md
