# Failure Mode: Premature Draft Promotion

## Название

**Преждевременное продвижение черновика** (Premature Draft Promotion)

## Описание

Черновик автоматически продвигается из стадии 2 (личный черновик, DS-my-strategy/drafts/) в стадию 3 (публичная заготовка, DS-Knowledge-Index status: draft) **без подтверждения пользователя**.

Конвейер творческого производства (PD.FORM.005) имеет критическое различение:

- **Черновик** — личный, для себя. Может быть сырым, незаконченным, с вопросами. Никому не показывается.
- **Заготовка** — публичная (видна в Knowledge Index). Должна быть связной и понятной единомышленникам.

**Граница черновик → заготовка = граница приватности.** Переход должен требовать явного решения пользователя, потому что:

1. **Концептуальная готовность** — пользователь должен осознанно оценить: «Готов ли я показать это?»
2. **Репутационный риск** — публикация сырого текста = потеря доверия читателя
3. **Когнитивная нагрузка** — автопромоут заставляет пользователя немедленно реагировать (откатывать), вместо завершения текста в своём темпе

## Симптомы

- Черновик появляется в DS-Knowledge-Index (status: draft) без явного действия пользователя
- Пользователь обнаруживает публично видимый черновик, который не готов к показу
- Необходимость отката (удаление из Knowledge Index, возврат в drafts/)
- Сломанное доверие к автоматизации («система решила за меня»)

## Контекст возникновения

**Trigger (исходное событие):**
- Черновик в DS-my-strategy/drafts/ достигает TTL (7 дней)
- Автоматический сценарий (note-review? session-close?) обнаруживает истечение TTL

**Проблема:** Сценарий интерпретирует истечение TTL как сигнал к **автоматическому продвижению**, хотя TTL должен быть сигналом к **принятию решения** (приоритизировать / продвинуть / закрыть), а не выполнению действия.

**Место в архитектуре:**
- PD.FORM.005 Creative Pipeline §6.1 определяет TTL для черновика (≤7 дней)
- §6.1 «При истечении»: «На стратегировании: приоритизировать / продвинуть / закрыть»
- **Ошибка:** автоматизация выбрала ветку «продвинуть» без согласования

## Почему это ошибка

1. **Bypass концептуального гейта** — переход черновик → заготовка требует оценки связности текста (PD.FORM.005 §3: «Тест: можно ли показать этому текст другому человеку?»)
2. **Нарушение автономии** — решение о публичности текста принимает сценарий, а не автор
3. **Несоответствие PD.FORM.005** — формализация конвейера требует принятия решения на стратегировании, а не автопромоут
4. **Двойная работа** — промоут + откат дороже, чем явное подтверждение перед промоутом

## Источник проблемы

**Гипотеза 1:** Сценарий note-review или session-close неправильно интерпретирует правило TTL:

```
# Неправильная интерпретация
if ttl_expired(draft) and not processed:
    promote_to_workpiece(draft)  # ❌ autopromote

# Правильная интерпретация
if ttl_expired(draft) and not processed:
    suggest_decision(draft)      # ✅ suggest to user
    # пользователь: приоритизировать / продвинуть / закрыть
```

**Гипотеза 2:** Отсутствует gate «подтверждение пользователя» между черновиком и заготовкой.

## Правильное поведение

| Стадия | TTL истёк | Действие сценария | Действие пользователя |
|--------|-----------|-------------------|----------------------|
| Заметка (fleeting-notes) | 7 дней | note-review маршрутизирует автоматически | Нет действия (маршрутизация = категоризация) |
| Черновик (drafts/) | 7 дней | **Предложить решение** на Strategy Session | Приоритизировать / продвинуть / закрыть |
| Заготовка (Knowledge Index draft) | 14 дней | Предложить решение на Strategy Session | Опубликовать / закрыть |

**Gate между черновиком и заготовкой:**

```yaml
transition:
  from: draft (stage 2)
  to: workpiece (stage 3)
  gate:
    type: user-confirmation
    prompt: "Черновик '{title}' готов к публичной видимости?"
    options:
      - promote (yes, move to Knowledge Index as draft)
      - prioritize (no, add to priority drafts)
      - archive (no, close with reason)
```

1. **Immediate fix:** Добавить gate «подтверждение пользователя» в сценарии note-review/session-close перед промоутом черновика
2. **Long-term fix:** Четко разделить:

3. **Audit:** Проверить все сценарии на наличие аналогичных autopromote без подтверждения
