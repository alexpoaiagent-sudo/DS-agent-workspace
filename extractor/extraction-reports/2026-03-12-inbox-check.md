```markdown
### DP.D.040: Интеллект = Качество убеждений ≠ Накопление информации

| Интеллект (качество убеждений) | Накопление информации |
|-------------------------------|----------------------|
| Качество убеждений + решений из них | Количество знаний |
| Изменение парадигм (мировоззрения) | Расширение базы данных |
| Компиляция знания (processing) | Сбор знания (storage) |
| Измеряется способностью действовать корректно | Измеряется объёмом запомненного |

**Почему важно**: Продукт для развития интеллекта (IWE) должен фокусироваться на изменении мировоззрения (компиляция знания в убеждения → качественные решения), а не на накоплении информации. Антипаттерн: мемы о саморазвитии, где успех = количество прочитанных книг / пройденных курсов (непродуктивные мемы). Продуктивный подход: качество убеждений → качество решений → качество жизни.

**Импликация для IWE**: Платформа должна менять мировоззрение созидателя (через экстракцию знаний → формализацию → интеграцию в Pack → изменение убеждений), а не просто предоставлять доступ к знанию (библиотека курсов).

**Тест**: Продукт меняет убеждения пользователя и качество его решений? Да → развивает интеллект. Продукт даёт доступ к информации без изменения убеждений? → библиотека (не развитие интеллекта).

> Связано: DP.ARCH.001 (характеристика «Обучаемость» — экзоскелет vs протез), DP.D.025 (Harness — среда определяет результат), DP.M.001 (Экстракция знаний — информация → знание)
```

**Вердикт:** accept
**Обоснование:** Различение легитимно и критически важно для позиционирования IWE как платформы развития интеллекта (не библиотеки контента). Определяет архитектурный принцип: система должна менять убеждения, а не просто накапливать информацию. Размещение в DP оправдано — это принцип проектирования платформы, а не личного развития (PACK-personal). Совместимо с существующими характеристиками (DP.ARCH.001: обучаемость, экзоскелет vs протез).
